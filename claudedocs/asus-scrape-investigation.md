# ASUS India scrape investigation

ASUS India does **not** use Magento like HP — `https://www.asus.com/in/laptops/` is a series-list page with filter chips (no real product cards), and `https://www.asus.com/in/store/laptops/` ships React-rendered cards whose prices are filled in by client-side JS, so a static `curl` only sees `₹` characters in promo banners (~9 hits) and 0 product prices. Confirmed via Playwright probing inside the worker container that the rendered DOM exposes 12 cards on `/in/store/laptops/` and 83 cards on `/in/store/`, each with class `ProductCardNormalGrid__productCardContainer__*`, an `<h2>` title, an `<a class="ProductCardNormalGrid__headingRow*">` link, and a `<div class="ProductCardNormalGrid__priceDiscount*">` price (e.g. `₹ 379,990.00`). Class hashes (`__1HpeZ`, `__3DUH_`) are webpack-generated, so I used `[class*="..."]` substring selectors so the config survives bundle rebuilds. Configured site_id=30 to use the existing `dynamic` (Playwright) scraper with `wait_selector` + `scroll_to_bottom` + `settle_ms=2500`, two category URLs (`/in/store/laptops/`, `/in/store/`), `enabled=true`. PATCH'd via the admin API, triggered a run, last_status=success, **81 ASUS listings now in the DB** (deduped from 12 + 83 by URL uniqueness — many cards on `/in/store/` were repeats of the laptops page), with valid prices spanning ₹56,990–₹431,990. No scraper code changes; ruff still clean.
